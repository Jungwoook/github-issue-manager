# 260420 Task

## 1. docs 01~12 현행화
요약: 현재 구현과 다른 문서 내용을 플랫폼 공통 구조 기준으로 정리

- 문제: 01~12 문서가 GitHub 전용 단계와 GitLab 계획 단계에 일부 머무름
- 지시사항: 현재 구현과 다른 부분 정정, 이미 진행한 내용은 기존 문서에 통합, 미진행 문서는 보존
- 작업: PRD, 아키텍처, API, 데이터 모델, 구현 요약, 유스케이스, PAT, 메인 플로우, 운영 설정, 플랫폼 추상화, GitHub/GitLab 비교, GitLab 계획 문서 갱신
- 결과: GitHub/GitLab 공통 구현 상태와 남은 후속 범위 분리

## 2. 플랫폼 공통화 완료 항목 반영
요약: 백엔드/프론트 공통화 완료 내용을 기존 문서에 통합

- 문제: PlatformGateway, PlatformConnection, platform API 경로, GitLab baseUrl 반영 상태가 문서별로 불일치
- 지시사항: 이미 완료된 항목은 계획이 아니라 현재 상태로 정리
- 작업: `/api/platforms/{platform}` 계약, `platform + externalId`, GitLab gateway, 프론트 platform 라우트 반영
- 결과: 문서 기준과 코드 구조 일치

## 3. 미진행 범위 보존
요약: 아직 구현하지 않은 기능은 삭제하지 않고 후속 범위로 분리

- 문제: 라벨/담당자/우선순위, self-managed 안정화, 접근 제어 보강이 완료 기능처럼 보일 위험
- 지시사항: 미진행 문서는 그대로 보존하되 현재 상태와 구분
- 작업: 제외 범위, 남은 후속 범위, Backend Follow-up 항목 재배치
- 결과: 완료/미완료 경계 명확화

## 4. 모듈화 피드백 반영
요약: 현재 구조를 완전한 모듈 구조가 아닌 어댑터 분리 단계로 재정의

- 문제: 새 플랫폼 추가 시 `PlatformType`, resolver, 프론트 metadata 등 공통 코드 수정 발생
- 지시사항: 확장성 한계를 문서에 명시하고 이후 모듈화 단계 추가
- 작업: 아키텍처, 구현 요약, 플랫폼 추상화 문서에 registry/capability/metadata 단계 반영
- 결과: 플랫폼 모듈화 전환 계획 문서화

## 5. 최소 데이터 모델 정리
요약: 현재 기능에 쓰지 않는 원격/캐시/연결 필드 제거

- 문제: GitHub subset 기준 현재 기능에 불필요한 필드가 DTO와 엔티티에 잔존
- 지시사항: 필요해질 때 다시 추가하는 방향으로 현재 불필요 필드 제거
- 작업: RemoteIssue.repositoryExternalId, RemoteComment.issueExternalId 제거
- 작업: RepositoryCache.defaultBranch, RepositoryCache.lastPushedAt 제거
- 작업: PlatformConnection.tokenScopes와 응답/프론트 타입 제거
- 결과: 원격 DTO와 저장 모델의 현재 기능 기준 최소화
