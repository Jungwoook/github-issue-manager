# 260427 작업 기록

## 1. 대표 흐름 다이어그램 표기 정리

요약: 14번 문서 다이어그램의 모듈 표기 통일

- 문제: 대표 흐름 다이어그램에 `*.api`가 노드명처럼 표시됨
- 지시사항: 다이어그램에서 `.api` 제거
- 작업: sequence participant를 실제 모듈명으로 변경
- 작업: 공개 API 호출 규칙은 설명 문장으로 분리
- 결과: 목표 의존 그래프와 대표 흐름 표기 일관성 확보

## 2. PAT 등록 기준 전환 계획 정리

요약: app 조립 방식 기준 후속 전환 계획 구체화

- 문제: PAT 등록 선택지가 비교 수준에 머물러 실제 전환 단계가 불명확
- 지시사항: A 선택지 기준 전환 계획 작성
- 작업: app -> platform 검증, app -> connection 저장 기준 명시
- 작업: platform credential API, app 조립, 업무 모듈 connection 의존 제거 단계 정리
- 결과: 순환 의존 방지 기준과 단계별 검증 포인트 확보

## 3. 전환 계획 단계 단순화

요약: 준비 작업 통합, 실행 단계 축소

- 문제: 준비성 작업이 별도 단계로 쪼개져 전환 계획이 장황함
- 지시사항: 단계를 단순화
- 작업: PAT 등록, 원격 호출 경계, Gradle/테스트, 문서 정리 4단계로 재구성
- 결과: 실행 흐름 중심의 후속 전환 계획 확보

## 4. 완료 후 구조 문서 작성 항목 추가

요약: 전환 완료 후 모듈/서비스 구조 문서화 명시

- 문제: 전환 결과를 설명하는 구조 문서 산출물이 단계에 명확히 없음
- 지시사항: 4단계에 구조 설명 문서 작성 추가
- 작업: 각 모듈과 서비스 구조 설명 문서 작성 항목 추가
- 결과: 전환 완료 후 리뷰와 유지보수 기준 확보
