# 260422 작업 기록

## 1. connection 모듈 2차 전환 계속

요약: app → connection.api 의존 경계 적용

- 문제: app 서비스가 connection 내부 `AuthService`와 entity/token 메서드에 직접 의존
- 지시사항: 2차 전환 방향으로 connection 모듈 분리 계속 진행
- 작업: `PlatformConnectionFacade` / `CurrentConnection` / `TokenAccess` 도입
- 작업: auth/user controller → connection facade 호출로 전환
- 작업: repository / issue / comment service → token access result 사용으로 전환
- 작업: 설계 문서 2차 적용 상태 갱신
- 결과: connection 내부 서비스 직접 주입 제거
- 결과: app 업무 서비스의 connection entity 직접 접근 제거
- 검증: `.\gradlew.bat clean :app:bootJar test` 성공

## 2. repository 모듈 3차 전환

요약: 저장소 캐시 책임을 repository 물리 모듈로 이동

- 문제: repository 책임이 app 내부에 남아 issue/comment가 repository entity와 service에 직접 의존
- 지시사항: 3차 전환 진행
- 작업: `RepositoryCache`, repository JPA, `RepositoryService`, repository DTO 이동
- 작업: `RepositoryFacade` / `RepositoryAccess` 도입
- 작업: app controller / issue / comment service → repository facade 호출로 전환
- 작업: sync 상태 기능을 `shared-kernel`로 이동
- 결과: app의 repository 내부 service/entity 직접 의존 제거
- 결과: repository 모듈 컴파일 경계 생성
- 검증: `.\gradlew.bat clean :app:bootJar test` 성공

- 결과: issue 모듈 컴파일 경계 생성
