# 260414 Task

## 1. 작업 기록 날짜 기준 정리
요약: 날짜 변경 시 새 task 파일 생성

- 문제: 날짜 변경 후 기존 파일 계속 사용
- 지시사항: 작업 당일 기준 새 task 파일 생성
- 작업: 날짜 기준 파일 생성 규칙 추가, 오늘 파일 생성
- 결과: 날짜별 작업 기록 분리 기준 확보

## 2. 프론트 리팩토링 영향 범위 정리
요약: 백엔드 구조 변경 기준 프론트 수정 범위 문서화

- 문제: 백엔드 3차 리팩토링 이후 프론트에서 어떤 변경이 필요한지 한눈에 보기 어려움
- 지시사항: 기능 변경보다 구조 변경 관점에서 프론트 영향 범위를 리팩토링 문서에 정리
- 작업: API 경로, DTO 필드명, 라우트, 쿼리 키, GitHub 전용 모듈 구조 영향 범위 정리
- 결과: 프론트 리팩토링 시작 전 필수 변경 범위와 권장 순서 문서 확보

## 3. 프론트 1차 리팩토링
요약: 새 백엔드 계약 반영, 기존 GitHub 화면 동작 유지

- 문제: 프론트 API 경로와 타입이 백엔드 3차 계약과 맞지 않아 화면 동작이 깨질 수 있음
- 지시사항: 프론트 1차 범위로 API 경로, DTO 필드명, 데이터 계층을 먼저 정리
- 작업: 플랫폼 경로 API 호출 반영, query key 정리, 저장소/이슈/댓글/토큰 화면을 새 필드 기준으로 수정
- 결과: 기존 GitHub 화면 동작 유지, 프론트 빌드 통과
