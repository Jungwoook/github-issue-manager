## 1. 공통 모듈 분리 방향 정리

### 지시사항

- GitHub 외 플랫폼 확장을 고려한 공통 모듈 분리 방향을 정리한다.

### 작업

- 현재 구조를 검토해 GitHub 전용 결합 지점을 정리했다.
- 공통 코어와 GitHub 어댑터로 나누는 방향을 기준으로 정리했다.

## 2. 공통 모듈 인터페이스 문서 작성

### 지시사항

- 공통 모듈 인터페이스 초안을 문서로 남긴다.

### 작업

- 공통 타입과 게이트웨이 중심의 인터페이스 초안을 문서화했다.
- 단계별 리팩토링 범위도 함께 정리했다.

## 3. 문서 상단 요약 추가

### 지시사항

- 문서 상단에 짧은 요약을 추가한다.

### 작업

- 문서 최상단에 핵심 방향만 빠르게 볼 수 있는 요약 섹션을 추가했다.

## 4. 프론트 리팩토링 방침 정리

### 지시사항

- 프론트는 백엔드 공통화 완료 후 한 번에 수정하는 방침으로 정리한다.

### 작업

- 프론트는 이번 단계에서 부분 수정하지 않고, 백엔드 구조 안정화 후 일괄 수정하는 방향으로 문서에 반영했다.

## 5. 작업 기록 문서 정리

### 지시사항

- 작업 기록 문서를 별도 폴더로 정리한다.

### 작업

- 작업 기록 전용 폴더를 만들고 기존 문서를 이동했다.
- 오늘 작업 기록도 같은 구조로 추가했다.
