# 260413 Task

## 1. 공통 모듈 분리 방향 정리

### 문제

- GitHub 전용 구조를 다른 플랫폼으로 확장하기 어려운 상태였다.

### 지시사항

- GitHub 외 플랫폼 확장을 고려한 공통 모듈 분리 방향을 정리한다.

### 작업

- 현재 구조를 검토하고 공통 코어와 GitHub 어댑터 방향으로 정리했다.

### 해결

- 플랫폼 공통화의 기준 구조와 우선순위를 먼저 합의할 수 있게 됐다.

## 2. 공통 모듈 인터페이스 문서 작성

### 문제

- 실제 리팩토링 전에 공통 인터페이스 기준이 문서로 필요했다.

### 지시사항

- 공통 모듈 인터페이스 초안을 문서로 남긴다.

### 작업

- 공통 타입, 게이트웨이, 단계별 리팩토링 범위를 문서화했다.

### 해결

- 이후 코드 작업에서 참고할 공통화 초안이 준비됐다.

## 3. 문서 상단 요약 추가

### 문제

- 문서 본문이 길어서 핵심 방향을 빠르게 파악하기 어려웠다.

### 지시사항

- 문서 상단에 짧은 요약을 추가한다.

### 작업

- 문서 최상단에 핵심 방향만 정리한 요약 섹션을 추가했다.

### 해결

- 문서를 열면 긴 본문 전에 핵심 방향을 바로 확인할 수 있게 됐다.

## 4. 프론트 리팩토링 방침 정리

### 문제

- 프론트를 중간 단계마다 수정하면 반복 변경이 커질 위험이 있었다.

### 지시사항

- 프론트는 백엔드 공통화 완료 후 한 번에 수정하는 방침으로 정리한다.

### 작업

- 프론트는 백엔드 구조 안정화 후 일괄 수정하는 방향으로 문서에 반영했다.

### 해결

- 공통화 작업 순서를 백엔드 우선으로 명확히 정리할 수 있게 됐다.

## 5. 작업 기록 문서 정리

### 문제

- 작업 기록 문서가 흩어져 있어 찾기와 관리가 불편했다.

### 지시사항

- 작업 기록 문서를 별도 폴더로 정리한다.

### 작업

- 작업 기록 전용 폴더를 만들고 기존 문서와 오늘 기록을 정리했다.

### 해결

- 작업 기록 위치와 형식이 한 곳으로 정리됐다.

## 6. 플랫폼 공통화 1차 리팩토링

### 문제

- GitHub API 타입 의존이 서비스 레이어에 직접 묶여 있어 공통화 시작점이 필요했다.

### 지시사항

- 10번 문서 기준의 1차 리팩토링을 진행한다.

### 작업

- 공통 플랫폼 포트와 원격 DTO를 추가하고, 서비스가 GitHub 구현 대신 공통 게이트웨이를 사용하도록 변경했다.
- 백엔드 테스트를 실행해 동작을 확인했다.

### 해결

- GitHub 동작은 유지한 채 백엔드 1차 공통화 기반이 생겼다.
