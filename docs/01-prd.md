# PRD

## 1. 프로젝트 개요

`github issue manager`는 사용자가 자신의 GitHub 저장소와 이슈를 조회하고 관리할 수 있도록 돕는 소규모 웹 애플리케이션이다.

현재 기준으로 이 시스템은 자체 저장소나 자체 이슈를 운영하지 않는다. GitHub 저장소, GitHub 이슈, GitHub 댓글을 원본 데이터로 사용하고, 애플리케이션 내부 DB는 사용자 정보, PAT 연결 정보, 캐시, 동기화 상태를 저장하는 용도로 사용한다.

## 2. 목표

### 2.1 제품 목표

- 사용자가 GitHub PAT를 등록할 수 있다.
- 사용자가 자신의 GitHub 저장소 목록을 조회할 수 있다.
- 사용자가 저장소별 GitHub 이슈 목록과 상세를 확인할 수 있다.
- 사용자가 GitHub 이슈를 생성, 수정, 닫기할 수 있다.
- 사용자가 GitHub 댓글을 조회하고 작성할 수 있다.
- 저장소, 이슈, 댓글을 캐시 기반으로 조회하고 필요 시 수동 새로고침할 수 있다.

### 2.2 현재 범위

- 개인 사용자 1명을 기준으로 구현한다.
- 인증 방식은 GitHub OAuth가 아니라 사용자 제공 PAT 기반이다.
- 라벨, 담당자, 우선순위, sub-issue 같은 확장 기능은 현재 범위에서 제외한다.

## 3. 핵심 사용자 문제

- GitHub 저장소와 이슈를 한 화면 흐름 안에서 빠르게 보고 싶다.
- 반복적으로 보는 저장소와 이슈를 매번 GitHub 웹에서 직접 찾는 것이 번거롭다.
- GitHub 원본 데이터를 그대로 사용하되, 조회 속도와 단순한 UI 흐름을 확보하고 싶다.

## 4. 주요 기능

### 4.1 PAT 연결

- GitHub Personal Access Token 등록
- 현재 연결 상태 확인
- PAT 연결 해제

### 4.2 저장소

- 내 GitHub 저장소 목록 조회
- 저장소 캐시 새로고침
- 저장소 선택 후 이슈 화면 이동

### 4.3 이슈

- 이슈 목록 조회
- 이슈 목록 새로고침
- 이슈 생성
- 이슈 상세 조회
- 이슈 수정
- 이슈 닫기
- 제목, 상태 기준 검색 및 필터링

### 4.4 댓글

- 댓글 목록 조회
- 댓글 새로고침
- 댓글 작성

## 5. 제외 범위

- GitHub App 기반 인증/설치 구조
- 다중 사용자 협업 시나리오
- 라벨 CRUD
- 담당자 선택
- 우선순위 관리
- sub-issue 구조
- 조직 단위 기능

## 6. 성공 기준

- 사용자가 PAT 등록 후 자신의 저장소 목록을 볼 수 있다.
- 저장소 선택 후 이슈 목록과 상세를 확인할 수 있다.
- 이슈 생성, 수정, 닫기, 댓글 작성이 GitHub에 반영된다.
- 저장소, 이슈, 댓글 각각 수동 새로고침이 가능하다.
