# PRD

## 1. 프로젝트 개요

`github issue manager`는 사용자가 외부 개발 플랫폼의 저장소와 이슈를 조회하고 관리할 수 있도록 돕는 소규모 웹 애플리케이션이다.

현재 구현은 GitHub를 기본 플랫폼으로 사용하고, GitLab을 두 번째 플랫폼으로 연결할 수 있는 공통 구조를 갖는다. 저장소, 이슈, 댓글은 외부 플랫폼을 원본 데이터로 사용하며, 애플리케이션 DB는 사용자 정보, 플랫폼 연결 정보, 캐시, 동기화 상태를 저장한다.

## 2. 목표

### 2.1 제품 목표

- 사용자가 플랫폼별 PAT를 등록할 수 있다.
- 사용자가 연결된 플랫폼의 저장소 또는 프로젝트 목록을 조회할 수 있다.
- 사용자가 저장소별 이슈 목록과 상세를 확인할 수 있다.
- 사용자가 이슈를 생성, 수정, 닫기할 수 있다.
- 사용자가 이슈 댓글을 조회하고 작성할 수 있다.
- 저장소, 이슈, 댓글을 캐시 기반으로 조회하고 필요 시 수동 새로고침할 수 있다.

### 2.2 현재 범위

- 개인 사용자 1명을 기준으로 구현한다.
- 인증 방식은 OAuth 또는 GitHub App이 아니라 사용자 제공 PAT 기반이다.
- 지원 플랫폼은 GitHub, GitLab이다.
- GitLab은 `baseUrl`을 저장하며, 기본값은 GitLab.com API이다.
- 라벨, 담당자, 우선순위, sub-issue 같은 확장 기능은 정식 백엔드 범위에서 제외한다.

## 3. 핵심 사용자 문제

- 여러 플랫폼의 저장소와 이슈를 같은 화면 흐름 안에서 빠르게 보고 싶다.
- 반복적으로 보는 저장소와 이슈를 매번 플랫폼 웹에서 직접 찾는 것이 번거롭다.
- 외부 플랫폼 원본 데이터를 그대로 사용하되, 조회 속도와 단순한 UI 흐름을 확보하고 싶다.

## 4. 주요 기능

### 4.1 플랫폼 연결

- GitHub PAT 등록
- GitLab PAT 등록
- GitLab base URL 입력 및 기본값 처리
- 현재 연결 상태 확인
- PAT 연결 해제

### 4.2 저장소

- 연결 플랫폼의 저장소/프로젝트 목록 조회
- 저장소 캐시 새로고침
- 저장소 선택 후 이슈 화면 이동

### 4.3 이슈

- 이슈 목록 조회
- 이슈 목록 새로고침
- 이슈 생성
- 이슈 상세 조회
- 이슈 수정
- 이슈 닫기
- 제목, 상태 기준 검색 및 필터링

### 4.4 댓글

- 댓글 목록 조회
- 댓글 새로고침
- 댓글 작성

## 5. 제외 범위

- OAuth 기반 인증
- GitHub App 기반 인증/설치 구조
- 다중 사용자 협업 시나리오
- 라벨 CRUD와 이슈 라벨 연결의 백엔드 구현
- 담당자 선택
- 우선순위 관리
- sub-issue 구조
- 조직 또는 그룹 단위 관리 기능
- GitLab merge request 연동

## 6. 성공 기준

- 사용자가 GitHub 또는 GitLab PAT 등록 후 저장소/프로젝트 목록을 볼 수 있다.
- 저장소 선택 후 이슈 목록과 상세를 확인할 수 있다.
- 이슈 생성, 수정, 닫기, 댓글 작성이 외부 플랫폼에 반영된다.
- 저장소, 이슈, 댓글 각각 수동 새로고침이 가능하다.
- 플랫폼별 API 경로와 응답 DTO가 공통 계약으로 동작한다.
