# PRD

## 1. 프로젝트 개요

`github issue manager`는 사용자가 자신의 외부 플랫폼 저장소와 이슈를 조회하고 관리할 수 있도록 돕는 소규모 웹 애플리케이션이다.

현재 실제 사용 흐름은 GitHub를 기본 플랫폼으로 한다. 다만 백엔드는 GitHub 전용 구현에 고정되지 않도록 platform / connection / repository / issue / comment 모듈로 책임을 분리했다.

## 2. 목표

### 2.1 제품 목표

- 사용자가 플랫폼 PAT를 등록할 수 있다.
- 사용자가 접근 가능한 저장소 목록을 조회할 수 있다.
- 사용자가 저장소별 이슈 목록과 상세를 확인할 수 있다.
- 사용자가 이슈를 생성, 수정, 닫기할 수 있다.
- 사용자가 이슈 댓글을 조회하고 작성할 수 있다.
- 저장소, 이슈, 댓글을 캐시 기반으로 조회하고 필요 시 수동 새로고침할 수 있다.

### 2.2 구조 목표

- 외부 플랫폼 API 의존성을 platform 모듈 뒤로 격리한다.
- PAT 저장, 암호화, 세션 연결 책임을 connection 모듈에 둔다.
- repository / issue / comment 모듈은 각자 캐시와 유스케이스를 소유한다.
- 업무 모듈은 token, baseUrl, GitHub/GitLab adapter 구현을 직접 알지 않는다.
- 모듈 경계와 의존 방향은 테스트로 검증한다.

## 3. 현재 범위

- 개인 사용자 1명을 기준으로 구현한다.
- 인증 방식은 OAuth/GitHub App이 아니라 사용자 제공 PAT 기반이다.
- 기본 플랫폼은 GitHub다.
- API 경로는 `/api/platforms/{platform}/...` 형식을 사용한다.
- GitLab adapter 구조는 준비되어 있지만, 포트폴리오 범위는 백엔드 모듈 경계와 플랫폼 의존성 격리에 둔다.

## 4. 핵심 사용자 문제

- 저장소와 이슈를 한 화면 흐름 안에서 빠르게 보고 싶다.
- 반복적으로 보는 이슈 데이터를 매번 외부 플랫폼 웹에서 직접 찾는 것이 번거롭다.
- 외부 플랫폼 원본 데이터를 유지하면서 조회 속도와 단순한 UI 흐름을 확보하고 싶다.

## 5. 주요 기능

### 5.1 플랫폼 연결

- 플랫폼 PAT 등록
- 현재 연결 상태 확인
- 플랫폼 연결 해제
- 로그아웃

### 5.2 저장소

- 저장소 목록 조회
- 저장소 캐시 새로고침
- 저장소 상세 조회
- 저장소 동기화 상태 조회

### 5.3 이슈

- 이슈 목록 조회
- 이슈 목록 새로고침
- 이슈 생성
- 이슈 상세 조회
- 이슈 수정
- 이슈 닫기
- 제목, 상태 기준 검색 및 필터링
- 이슈 동기화 상태 조회

### 5.4 댓글

- 댓글 목록 조회
- 댓글 새로고침
- 댓글 작성

## 6. 제외 범위

- OAuth / GitHub App 기반 인증
- 다중 사용자 협업 시나리오
- 라벨 CRUD
- 담당자 선택
- 우선순위 관리
- 마일스톤
- sub-issue 구조
- 조직 단위 기능

## 7. 성공 기준

- 사용자가 PAT 등록 후 자신의 저장소 목록을 볼 수 있다.
- 저장소 선택 후 이슈 목록과 상세를 확인할 수 있다.
- 이슈 생성, 수정, 닫기, 댓글 작성이 외부 플랫폼에 반영된다.
- 저장소, 이슈, 댓글 각각 수동 새로고침이 가능하다.
- 백엔드 모듈 간 의존 방향이 테스트로 검증된다.
