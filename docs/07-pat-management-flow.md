# PAT 관리 흐름

## 목적

- 사용자가 입력한 GitHub PAT가 어디서 검증되고 저장되며 이후 어떻게 사용되는지 정리한다.

## 현재 흐름

1. 프론트에서 PAT를 입력한다.
2. 백엔드가 GitHub 사용자 API로 토큰을 검증한다.
3. 검증된 PAT는 암호화되어 저장된다.
4. 이후 저장소, 이슈, 댓글 조회 시 복호화 후 GitHub API 호출에 사용된다.

## 관리 기준

- PAT는 영구 저장소에 평문으로 보관하지 않는다.
- 운영 환경에서는 HTTPS를 기본 전제로 유지한다.
- 요청 본문, 예외 로그, 프록시 로그에 PAT가 남지 않도록 계속 점검한다.
- 암호화 키는 운영 전용 값으로 별도 관리한다.
