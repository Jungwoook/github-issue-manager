# Core Use Cases

## 1. 개요

현재 유스케이스는 PAT 기반 개인 사용자 흐름을 기준으로 한다.

- 시스템은 GitHub 원본 데이터를 사용한다.
- 사용자는 자신의 PAT로 접근 가능한 저장소와 이슈만 다룬다.
- 현재 범위는 저장소, 이슈, 댓글, 상태 변경까지다.

## 2. 액터

- `방문자`: 아직 PAT를 등록하지 않은 사용자
- `사용자`: PAT 등록을 완료하고 기능을 사용할 수 있는 사용자

## 3. 유스케이스 목록

- UC-01 PAT 등록
- UC-02 저장소 목록 조회
- UC-03 저장소 새로고침
- UC-04 이슈 목록 조회
- UC-05 이슈 새로고침
- UC-06 이슈 생성
- UC-07 이슈 상세 조회
- UC-08 이슈 수정
- UC-09 이슈 닫기
- UC-10 댓글 조회
- UC-11 댓글 새로고침
- UC-12 댓글 작성

## 4. 주요 유스케이스

### UC-01 PAT 등록

- 목적: 사용자가 GitHub PAT를 등록해 시스템을 사용할 수 있게 한다.
- 액터: 방문자
- 기본 흐름:
1. 사용자가 PAT 설정 화면에 진입한다.
2. 사용자가 PAT를 입력한다.
3. 시스템이 GitHub `/user` 호출로 PAT를 검증한다.
4. 시스템이 내부 사용자와 GitHub 계정 정보를 저장한다.
5. 사용자는 저장소 화면으로 이동한다.

### UC-02 저장소 목록 조회

- 목적: 사용자가 접근 가능한 GitHub 저장소 목록을 확인한다.
- 액터: 사용자
- 기본 흐름:
1. 사용자가 저장소 화면에 진입한다.
2. 시스템이 저장소 캐시를 조회한다.
3. 캐시된 저장소 목록을 화면에 표시한다.

### UC-03 저장소 새로고침

- 목적: 사용자가 GitHub 저장소 목록을 다시 동기화한다.
- 액터: 사용자
- 기본 흐름:
1. 사용자가 저장소 새로고침 버튼을 누른다.
2. 시스템이 GitHub 저장소 목록을 조회한다.
3. 저장소 캐시를 갱신한다.
4. 갱신된 저장소 목록을 표시한다.

### UC-04 이슈 목록 조회

- 목적: 사용자가 선택한 저장소의 이슈 목록을 확인한다.
- 액터: 사용자
- 기본 흐름:
1. 사용자가 특정 저장소를 선택한다.
2. 시스템이 이슈 캐시를 조회한다.
3. 제목과 상태 기준으로 필터링된 결과를 표시한다.

### UC-05 이슈 새로고침

- 목적: 사용자가 저장소의 최신 이슈 목록을 다시 불러온다.
- 액터: 사용자
- 기본 흐름:
1. 사용자가 이슈 새로고침 버튼을 누른다.
2. 시스템이 GitHub 이슈 목록을 조회한다.
3. 이슈 캐시를 갱신한다.
4. 갱신된 이슈 목록을 표시한다.

### UC-06 이슈 생성

- 목적: 사용자가 새 GitHub 이슈를 만든다.
- 액터: 사용자
- 기본 흐름:
1. 사용자가 제목과 설명을 입력한다.
2. 시스템이 GitHub에 이슈 생성을 요청한다.
3. 생성 결과를 이슈 캐시에 반영한다.
4. 생성된 이슈 상세 화면으로 이동한다.

### UC-07 이슈 상세 조회

- 목적: 사용자가 특정 이슈의 상세 정보를 확인한다.
- 액터: 사용자
- 기본 흐름:
1. 사용자가 이슈 상세 화면에 진입한다.
2. 시스템이 이슈 상세와 댓글 캐시를 조회한다.
3. 화면에 상태, 본문, 댓글 목록을 표시한다.

### UC-08 이슈 수정

- 목적: 사용자가 이슈 제목과 설명을 수정한다.
- 액터: 사용자
- 기본 흐름:
1. 사용자가 수정 화면에 진입한다.
2. 제목과 설명을 수정한다.
3. 시스템이 GitHub 이슈 수정 API를 호출한다.
4. 결과를 캐시에 반영한다.

### UC-09 이슈 닫기

- 목적: 사용자가 이슈를 닫는다.
- 액터: 사용자
- 기본 흐름:
1. 사용자가 이슈 닫기 동작을 실행한다.
2. 시스템이 GitHub 이슈 상태를 `CLOSED`로 변경한다.
3. 이슈 캐시를 갱신한다.
