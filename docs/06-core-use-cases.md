# Core Use Cases

## 1. 개요

현재 유스케이스는 PAT 기반 개인 사용자 흐름을 기준으로 한다.

- 사용자는 GitHub 또는 GitLab 중 하나의 플랫폼을 선택해 PAT를 등록한다.
- 시스템은 외부 플랫폼의 저장소/프로젝트, 이슈, 댓글을 원본으로 사용한다.
- 조회는 캐시를 우선 사용하고, 최신화는 수동 새로고침으로 수행한다.

## 2. 액터

- `방문자`: 아직 플랫폼 PAT를 등록하지 않은 사용자
- `사용자`: 플랫폼 PAT 등록을 완료하고 기능을 사용할 수 있는 사용자

## 3. 유스케이스 목록

- UC-01 플랫폼 PAT 등록
- UC-02 저장소/프로젝트 목록 조회
- UC-03 저장소/프로젝트 새로고침
- UC-04 이슈 목록 조회
- UC-05 이슈 새로고침
- UC-06 이슈 생성
- UC-07 이슈 상세 조회
- UC-08 이슈 수정
- UC-09 이슈 닫기
- UC-10 댓글 조회
- UC-11 댓글 새로고침
- UC-12 댓글 작성

## 4. 주요 유스케이스

### UC-01 플랫폼 PAT 등록

- 목적: 사용자가 GitHub 또는 GitLab PAT를 등록해 시스템을 사용할 수 있게 한다.
- 액터: 방문자
- 기본 흐름:
1. 사용자가 플랫폼 연결 화면에 진입한다.
2. 사용자가 플랫폼을 선택한다.
3. 사용자가 PAT를 입력한다.
4. GitLab인 경우 필요 시 base URL을 입력한다.
5. 시스템이 플랫폼 gateway로 현재 사용자 API를 호출해 PAT를 검증한다.
6. 시스템이 사용자와 플랫폼 연결 정보를 저장한다.
7. 사용자는 저장소 화면으로 이동한다.

### UC-02 저장소/프로젝트 목록 조회

- 목적: 사용자가 접근 가능한 저장소/프로젝트 목록을 확인한다.
- 액터: 사용자
- 기본 흐름:
1. 사용자가 저장소 화면에 진입한다.
2. 시스템이 현재 플랫폼과 세션 사용자를 확인한다.
3. 시스템이 저장소 캐시를 조회한다.
4. 화면에 저장소/프로젝트 목록을 표시한다.

### UC-03 저장소/프로젝트 새로고침

- 목적: 사용자가 외부 플랫폼의 최신 저장소/프로젝트 목록을 다시 동기화한다.
- 액터: 사용자
- 기본 흐름:
1. 사용자가 새로고침 버튼을 누른다.
2. 시스템이 플랫폼 gateway로 목록을 조회한다.
3. 저장소 캐시를 갱신한다.
4. 갱신된 목록을 표시한다.

### UC-04 이슈 목록 조회

- 목적: 사용자가 선택한 저장소의 이슈 목록을 확인한다.
- 액터: 사용자
- 기본 흐름:
1. 사용자가 저장소를 선택한다.
2. 시스템이 저장소 접근 가능 여부를 확인한다.
3. 시스템이 이슈 캐시를 조회한다.
4. 제목과 상태 기준으로 필터링된 결과를 표시한다.

### UC-05 이슈 새로고침

- 목적: 사용자가 선택한 저장소의 최신 이슈 목록을 다시 동기화한다.
- 액터: 사용자
- 기본 흐름:
1. 사용자가 이슈 새로고침 버튼을 누른다.
2. 시스템이 플랫폼 gateway로 이슈 목록을 조회한다.
3. 이슈 캐시를 갱신한다.
4. 갱신된 이슈 목록을 표시한다.

### UC-06 이슈 생성

- 목적: 사용자가 외부 플랫폼에 새 이슈를 만든다.
- 액터: 사용자
- 기본 흐름:
1. 사용자가 제목과 설명을 입력한다.
2. 시스템이 플랫폼 gateway로 이슈 생성 API를 호출한다.
3. 생성 결과를 이슈 캐시에 반영한다.
4. 생성된 이슈 상세 화면으로 이동한다.

### UC-07 이슈 상세 조회

- 목적: 사용자가 특정 이슈의 상세 정보를 확인한다.
- 액터: 사용자
- 기본 흐름:
1. 사용자가 이슈 상세 화면에 진입한다.
2. 시스템이 이슈 캐시를 조회한다.
3. 시스템이 댓글 캐시를 조회한다.
4. 화면에 상태, 본문, 댓글 목록을 표시한다.

### UC-08 이슈 수정

- 목적: 사용자가 이슈 제목, 설명, 상태를 수정한다.
- 액터: 사용자

2. 제목, 설명, 상태를 수정한다.
3. 시스템이 플랫폼 gateway로 이슈 수정 API를 호출한다.
4. 결과를 캐시에 반영한다.

### UC-09 이슈 닫기

- 목적: 사용자가 이슈를 닫는다.
- 액터: 사용자
- 기본 흐름:
1. 사용자가 이슈 닫기 동작을 실행한다.
2. 시스템이 외부 플랫폼 이슈 상태를 `CLOSED`로 변경한다.
3. 이슈 캐시를 갱신한다.
